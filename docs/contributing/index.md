# Contributor Overview

Thank you for your interest in contributing to the ServiceNow SDK for Go! Whether you're fixing a bug, improving documentation, or adding a new feature, your contributions are highly valued.

## How You Can Help

- **Code**: Fix bugs, add new APIs, or optimize the existing code.
- **Docs**: Improve our documentation, add more examples, or fix typos.
- **Issues**: Report bugs or suggest new features in the issue tracker.
- **Tests**: Add more test cases or improve our testing infrastructure.

## Where to Start?

1.  [**Development Setup**](setup.md): Get your environment ready.
2.  [**Architecture**](architecture.md): Understand how the SDK is built.
3.  [**Testing Guide**](testing.md): Learn how we ensure quality.
4.  **Issue Tracker**: Check for "good first issues" on GitHub.

## Our Philosophy

- **Simplicity**: We prefer simple, readable code over clever hacks.
- **Consistency**: We follow Go's standard conventions and the Kiota patterns.
- **Tests**: Every new feature or fix should come with tests.
- **Documentation**: Keep the documentation up to date with code changes.

Let's build a great SDK together!
